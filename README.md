# 楽天ROOMトラッカー（公開版）

楽天ROOMのフォロワー数・投稿数・いいね数を毎日自動で記録し、GASのWebアプリでグラフ表示するダッシュボードです。

## 構成

```
Code.js           — GAS バックエンド（自動取得・スプレッドシート書込・ダッシュボードのHTML配信）
Index.html        — ダッシュボード UI（jsDelivr経由で毎回最新版が配信される）
appsscript.json   — GAS マニフェスト
```

`Code.js`は各ユーザーのコピーの中で完結し、ROOMのデータや設定はそこから外に出ません。`Index.html`（UI）だけは、このリポジトリから毎回最新版を取得して表示する仕組みになっているため、開発者側でUIやダッシュボードの機能を更新すると、既にコピー済みのユーザーにも自動的に反映されます（jsDelivrのキャッシュにより、反映まで最大12時間ほどかかることがあります）。

## 特徴

- データ取得は完全自動（毎日決まった時刻にGASが自分のROOMプロフィールを取得・記録）
- スマホアプリのインストールや、iOSショートカットの作成は不要
- 初期セットアップだけはPCでの作業を推奨します（Apps Scriptの画面がスマホでは操作しにくいため）。セットアップ後の日常利用（ダッシュボードを見る）はスマホで問題ありません

## セットアップ手順（PC推奨）

スマホのみで作業する場合、画面が正しく表示されない・拡張機能メニューが出ない時は、ブラウザのメニュー（Safariなら「aA」、Chromeなら「⋮」）から「デスクトップ用Webサイトを表示」をタップしてください。セットアップの間だけ使う一時的な切り替えで十分です。

### 1. テンプレートをコピーする

以下のテンプレートのリンク末尾を `/copy` にしたURLを開くと、自動的に「コピーを作成」の確認画面が表示されます。「コピーを作成」を押すと、あなたのGoogleドライブに複製されます（コード一式も一緒にコピーされます）。コピーを開いた時点で「設定」シートが自動的に用意されます。

### 2. 自動取得を開始する

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く
2. 何も選び直さず、そのまま▶（実行）ボタンを押す（`setup`が先頭の関数として選ばれています）
3. 初回のみ権限確認画面が出るので、「権限を確認」→ Googleアカウントを選択 →「詳細」→「(プロジェクト名)に移動（安全ではないページ）」→「許可」と進む
4. 実行が完了すると、毎日指定した時刻にROOMプロフィールを取得・記録する時間主導トリガーが設定されます（デフォルトは毎日7時。変更したい場合は「設定」シートの`fetch_hour`を編集してから再度`setup`を実行）

この時点で自動収集は動き始めます。ダッシュボードを見るには、続けて手順3を行ってください。

### 3. ダッシュボードを見られるようにする（デプロイ）

1. Apps Scriptエディタ右上の「デプロイ」→「新しいデプロイ」
2. 種類の選択（歯車アイコン）→「ウェブアプリ」
3. 「アクセスできるユーザー」を「全員」にして「デプロイ」
4. 表示されたウェブアプリのURLをコピー

このURLは一度発行すれば以後もずっと使えます。以降`Code.js`側にほとんど変更を加える予定がないため、基本的にこの作業は最初の1回だけで済みます。

### 4. ROOM URLを入力してはじめる

手順3で発行されたダッシュボードURLをブラウザで開きます。初回は「あなたのROOM URLを入力してください」という画面が出るので、あなたの楽天ROOMプロフィールURL（例: `https://room.rakuten.co.jp/room_xxx`）を入力して「保存してはじめる」を押します。これでダッシュボードが表示され、以降は毎日自動で記録されます。スマホのホーム画面に追加しておくと、日々の確認が楽になります。

## データスキーマ

スプレッドシートの「記録」シートに以下の形式で記録されます（1日1回、同日は上書き）。

| 日時 | URL | フォロワー数 | フォロー数 | 投稿数 | いいね数 | ランク |
|------|-----|------------|----------|-------|--------|------|

## トラブルシューティング

**ダッシュボードが表示されない・データが記録されない**
- 「設定」シートの `room_url` が正しく入力されているか確認してください
- `setup` を再実行すると、トリガーが作り直されます

**「権限を確認」の画面で不安に感じる**
- このスクリプトは自分のコピーしたスプレッドシートに対してのみ動作し、外部に送信されるのはあなたが入力したROOM URLへのアクセスのみです。「Google で確認されていないアプリ」という警告は、個人が作成・複製したスクリプトでは通常表示されるもので、異常ではありません

**ダッシュボードの表示が急におかしくなった**
- Index.htmlはjsDelivr経由で毎回最新版を取得しています。一時的な取得失敗の場合は、少し時間を置いてから開き直してください
