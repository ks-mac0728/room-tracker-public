# 楽天ROOMトラッカー（公開版）

楽天ROOMのフォロワー数・投稿数・いいね数を記録し、GASのWebアプリでグラフ表示するダッシュボードです。

## 構成

```
Code.js            — GAS バックエンド（データ受信・スプレッドシート書込・ダッシュボードのHTML配信）
Index.html         — ダッシュボード UI（jsDelivr経由で毎回最新版が配信される）
ios_shortcut_js.js — iOSショートカットの中で実行するJS（ROOMページからのデータ抽出）
appsscript.json    — GAS マニフェスト
```

`Code.js`は各ユーザーのコピーの中で完結し、ROOMのデータや設定はそこから外に出ません。`Index.html`（UI）だけは、このリポジトリから毎回最新版を取得して表示する仕組みになっているため、開発者側でUIやダッシュボードの機能を更新すると、既にコピー済みのユーザーにも自動的に反映されます（jsDelivrのキャッシュにより、反映まで最大12時間ほどかかることがあります）。

## 特徴・大事な注意点

- **データ取得は「本物のSafari」で行います**。GASのサーバーから直接ROOMのページへアクセスしようとすると、楽天側のボット対策（Akamai）によって確実にブロックされることを検証済みのため、この方式は採用していません
- そのため、**完全に放置した自動更新はできません**。1日1回、通知が来たらタップして実行する（iOSショートカットの「オートメーション」機能を利用）運用になります。タップ自体はSafariでページを開くだけで、中身の送信は全部自動です
- 初期セットアップだけはPCでの作業を推奨します（Apps Scriptの画面がスマホでは操作しにくいため）。セットアップ後の日常利用（ショートカットの実行・ダッシュボードを見る）はスマホで完結します

## セットアップ手順（PC推奨）

スマホのみで作業する場合、画面が正しく表示されない・拡張機能メニューが出ない時は、ブラウザのメニュー（Safariなら「aA」、Chromeなら「⋮」）から「デスクトップ用Webサイトを表示」をタップしてください。セットアップの間だけ使う一時的な切り替えで十分です。

### 1. テンプレートをコピーする

以下のテンプレートのリンク末尾を `/copy` にしたURLを開くと、自動的に「コピーを作成」の確認画面が表示されます。「コピーを作成」を押すと、あなたのGoogleドライブに複製されます（コード一式も一緒にコピーされます）。コピーを開いた時点で「設定」シートが自動的に用意されます。

### 2. ダッシュボードを見られるようにする（デプロイ）

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く
2. 右上の「デプロイ」→「新しいデプロイ」
3. 種類の選択（歯車アイコン）→「ウェブアプリ」
4. 「アクセスできるユーザー」を「全員」にして「デプロイ」
5. 初回のみ権限確認画面が出るので、「権限を確認」→ Googleアカウントを選択 →「詳細」→「(プロジェクト名)に移動（安全ではないページ）」→「許可」と進む
6. 表示されたウェブアプリのURLをコピー

このURLは一度発行すれば以後もずっと使えます。以降`Code.js`側にほとんど変更を加える予定がないため、基本的にこの作業は最初の1回だけで済みます。

### 3. ROOM URLを入力してはじめる

手順2で発行されたダッシュボードURLをブラウザで開きます。初回は「あなたのROOM URLを入力してください」という画面が出るので、あなたの楽天ROOMプロフィールURL（例: `https://room.rakuten.co.jp/room_xxx`）を入力して「保存してはじめる」を押します。この時点ではまだグラフは空です（データはこの後のiOSショートカットから届きます）。

### 4. iOSショートカットを作成する

「ショートカット」アプリで、**名前を「楽天ROOMトラッカー」**にして、以下のアクションをこの順番で組み合わせます（名前はダッシュボードの「▶ 記録する」ボタンから直接起動するために使うので、変更する場合は手順5も参照してください）。

1. **Safariでページを開く** — URL: あなたの楽天ROOMプロフィールURL
2. **少し待つ** — 1〜2秒
3. **Webページ上でJavaScriptを実行** — `ios_shortcut_js.js` の内容をそのまま貼り付け
4. **テキストを辞書に変換** — 手順3の結果を辞書（JSON）に変換
5. **URLの内容を取得**
   - URL: 手順2でメモしたウェブアプリURL
   - 方式: POST
   - ヘッダー: `Content-Type: application/json`
   - 本文（JSON）:
     ```json
     {
       "action": "receiveStats",
       "url": 「辞書」の「url」,
       "followers": 「辞書」の「followers」,
       "following": 「辞書」の「following」,
       "posts": 「辞書」の「posts」,
       "likes": 「辞書」の「likes」,
       "rank": 「辞書」の「rank」
     }
     ```

ダッシュボード上部の「▶ 記録する」ボタンは、`shortcuts://run-shortcut?name=楽天ROOMトラッカー` というリンクになっており、タップするとショートカットアプリが開いて自動的にこのショートカットが実行されます（初回のみ「shortcutsで開きますか？」という確認が出ます）。ショートカット名を変える場合は、`Index.html`内の`run-shortcut-btn`のリンクの`name=`部分も同じ名前に変更してください。実行後は、ブラウザのダッシュボードのタブに戻って「↻ 更新」を押すと、記録した内容が反映されます。

### 5. 毎日の実行を促す通知を設定する（完全自動化はできません）

ショートカットの「オートメーション」タブで「個人用オートメーションを作成」→「時刻」を選び、毎日実行したい時刻を設定します。**「実行前に尋ねる」はオンのままにしてください**（これをオフにしてロック中に自動実行させようとしても、iOSの仕様上Safari・JS実行は動作しないことを確認済みです）。設定した時刻になると通知が届くので、タップして実行するだけで記録が完了します。

ホーム画面にダッシュボードのURLを追加しておくと、日々の確認が楽になります。

## データスキーマ

スプレッドシートの「記録」シートに以下の形式で記録されます（1日1回、同日は上書き）。

| 日時 | URL | フォロワー数 | フォロー数 | 投稿数 | いいね数 | ランク |
|------|-----|------------|----------|-------|--------|------|

## セキュリティ設定（任意・推奨）

スプレッドシートの「設定」シートに以下を追加すると、自分以外からのデータ送信を防げます。

| キー | 値 |
|------|----|
| api_secret_key | （任意の文字列。例: mysecret123） |

設定した場合は、iOSショートカットのPOST本文にも `"api_key": "（設定した文字列）"` を追加してください。

## トラブルシューティング

**ダッシュボードにグラフが出ない**
- iOSショートカットをまだ一度も実行していない可能性があります。手動で一度実行してみてください
- 「設定」シートの `room_url` が正しく入力されているか確認してください

**ショートカット実行後もデータが増えない**
- POST先のURLが、手順2で発行したウェブアプリURLと一致しているか確認してください
- `api_secret_key` を設定した場合、ショートカット側の `api_key` と一致しているか確認してください

**「権限を確認」の画面で不安に感じる**
- このスクリプトは自分のコピーしたスプレッドシートに対してのみ動作します。「Google で確認されていないアプリ」という警告は、個人が作成・複製したスクリプトでは通常表示されるもので、異常ではありません

**ダッシュボードの表示が急におかしくなった**
- Index.htmlはjsDelivr経由で毎回最新版を取得しています。一時的な取得失敗の場合は、少し時間を置いてから開き直してください

**Chromeでは使えますか？**
- データ送信はSafari専用です（「Webページ上でJavaScriptを実行」がSafariのページに対して動作するため）。ダッシュボードの閲覧自体はどのブラウザからでも可能です
